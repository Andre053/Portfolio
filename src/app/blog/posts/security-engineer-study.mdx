## Overview
This post outlines some of the key skills and theory that underpins a career in cybersecurity engineering. 
I also outline my current certificaiton path and planned certifications that would support such a role. 

## Potential cybersecurity roles
The cybersecurity profession is wide with many diverse roles. Not only does the job change depending on the organization you are supporting, there are
many different areas of speciality. 

Sophos (2026) outlines 8 key responsibilities of a cybersecurity specialist:
1. Risk assessment and vulnerability management
2. Security implementation
3. Monitoring and incident response
4. Security audits and compliance
5. Employee training and awareness
6. Documentation and reporting
7. Threat intelligence analysis
8. Security policy development 

Some common roles include:
- Threat Intelligence Analyst
- Cyber Security Analyst
- Threat Hunter
- Malware Researcher
- Cybersecurty Risk Advisor
- Incident Response Consultant
- Vulnerability Management Engineer
- Penetration Testing Consultant
- Security Automation Researcher
- Security Playbook Engineer 

While there is post-secondary education in cybersecurity, certifications and experience are often the route towards landing a role. 
Since cybersecurity is a subset of IT, many IT and computer science professionals may become exposed to the discipline and learn more
about it as they grow in their initial profession. 

## What does a security engineer do?
Indeed (2026) describes security engineers as technical professionals focused on implementing security measures. Security measures are any steps taken 
to improve the security posture of an organization or individual. 

Since understanding the plethora of security measures available and being able to implement them is key a 
base of cybersecurty expertise, I find this is an ideal role to work towards. 

## Security engineer skills
To be a successful security engineer, you need a strong understanding of the domain you are working in and the cybersecurity principles and 
frameworks that can inform security decisions. 

1. Understand the needs of and systems used in the organization
2. Understand concepts and frameworks to guide cybersecurity efforts
3. Be able to technically implement the proper security measures

Major areas or security:
1. Identity and access 
2. Network
3. Application 
4. Endpoint
5. Physical

Further, there are three different teams that can support cybersecurity efforts:
- Blue team (defensive)
- Red team (offensive)
- Purple team (attacks and defends)

## Cloud security engineer

While different organizations have different needs and corresponding systems, there are some common technologies that have become
the basis for whole careers. Where enterprise networks used to be the primary security posture of an organization (and thus where cybersecurity
was focused), the modern organization network has changed. Since the advent of cloud services, like those offered by AWS and Azure, nearly all 
organizations have adopted cloud services in some form. Due to the prevelance of cloud technologies and the importance it has to the data that
organizations must protect, the cloud has become a dominant area of cybersecurity need. 

Security engineering in the cloud requires a basis in cloud engineering, requiring skills such as the following:
- Scripting
- Coding
- Security
- IaC
- DevOps, CI/CD
- Systems thinking
- Architectural thinking
- Business mindset
