## What is the Associate Data Practitioner Certification
The Associate Data Practitioner (ADP) is a entry level GCP certification for data engineers. It is oriented towards a professional who uses GCP for data tasks, such as:
- Data ingestion
- Data transformation
- Data pipeline management
- Data analysis
- Machine learning
- Data visualziation

A basic understanding of cloud computing concepts is encouraged, such as IaaS, PaaS, and SaaS. The exam assesses the professionals ability to:
- Manage data
- Orchestrate data pipelines
- Prepare and ingest data
- Analyze and present data

About the exam:
- 2 hour time limit
- 50-60 multiple choice and multiple select questions
- $125 registration fee

## Exam guide overview
There are four sections of the exam:
1. Data preparation and ingestion (30%)
2. Data analysis and presentation (27%)
3. Data pipeline orchestration (18%)
4. Data management (25%)

These are further broken down into subsections:

**Data preparation and ingestion**
1. Prepare and process data
2. Extract and load data into appropriate Google Cloud storage systems

**Data analysis and presentation**
1. Identify data trends, patterns, and insights using BigQuery and Jupyter notebooks
2. Visualize data and create dashboards in Looker, given business requirements
3. Define, train, evaluate, and use ML models

**Data pipeline orchestration**
1. Design and implement simple data pipelines
2. Schedule, automate, and monitor basic data processing tasks

**Data management**
1. Configure access control and governance
2. Configure lifecycle management
3. Identify high availability and disaster recovery strategies for data in Cloud Storage and Cloud SQL
4. Apply security measures and ensure compliance with data privacy regulations

Concepts to understand:
- Data manipulation methodologies: ETL, ELT, ETLT
- Data transfer types
- Data quality and cleaning
- Data formats: CSV, JSON, APache Parquet, Apache Avro, structured db tables
- Data storage locations: Regional, dual-regional, multi-regional, zonal
- Data use cases by data requirements: Structured, unstructured, semi-structured
- ML project planning: Data collection, model training, model evaluation, prediction
- SQL for creating, training, and evaluating models
- Data pipeline types
- Data access control and governance: Least privileged
- Backup and recovery solutions, replication
- Encryption in use, in transit, and at rest

Tools to understand:
- Data cleaning: Cloud Data Fusion, BigQuery, SQL, Dataflow
- Data extraction: Dataflow, BigQuery Data Transfer Service, Database Migration Service, Cloud Data Fusion
- Data storage: Cloud Storage, BigQuery, Cloud SQL, Firestore, Bigtable, Spanner
- Data loading: gcloud and BQ CLI, Storage Transfer Service, BigQuery Data Transfer Service, client libraries, Pub/Sub
- Data analysis and visualization: BigQuery, Colab Enterprise (Jupyter notebooks), Looker/Looker Studio/LookML
- Machine learning: BigQuery ML training, evaluation, and inference; Auto ML; pretrained BigQuery; Model Registry
- Data transformation: Dataproc, Dataflow, Cloud Data Fusion, Cloud Composer, Dataform
- Data pipeline automation: BigQuery, Cloud Scheduler, Cloud Composer; Cloud Logging, Cloud Monitoring
- Data orchestration: Cloud Composer, scheduled queries, Dataproc Workflow Templates, Workflows 
- Eventarc triggers in event-driven pipelines
- Data governance: Identity and Access Management (IAM) roles, Cloud Storage access control, Analytics Hub data sharing
- Data lifecycle management: Cloud Storage classes based on frequency and retention, automated data removal with BigQuery, Cloud Storage
- Primary and secondary data storage locations for data redundancy
- Data privacy: customer-managed encryption keys (CMEK), customer-supplied encryption keys (CSEK), Google-managed encryption keys (GMEK); Cloud Key Management Service (Cloud KMS) to manage keys
