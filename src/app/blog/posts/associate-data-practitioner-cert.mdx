## What is the Associate Data Practitioner Certification
The Associate Data Practitioner (ADP) is a entry level GCP certification for data engineers. It is oriented towards a professional who uses GCP for data tasks, such as:
- Data ingestion
- Data transformation
- Data pipeline management
- Data analysis
- Machine learning
- Data visualziation

A basic understanding of cloud computing concepts is encouraged, such as IaaS, PaaS, and SaaS. The exam assesses the professionals ability to:
- Manage data
- Orchestrate data pipelines
- Prepare and ingest data
- Analyze and present data

About the exam:
- 2 hour time limit
- 50-60 multiple choice and multiple select questions
- $125 registration fee

## Exam guide overview
There are four sections of the exam:
1. Data preparation and ingestion (30%)
2. Data analysis and presentation (27%)
3. Data pipeline orchestration (18%)
4. Data management (25%)

These are further broken down into subsections:

**Data preparation and ingestion**
1. Prepare and process data
2. Extract and load data into appropriate Google Cloud storage systems

**Data analysis and presentation**
1. Identify data trends, patterns, and insights using BigQuery and Jupyter notebooks
2. Visualize data and create dashboards in Looker, given business requirements
3. Define, train, evaluate, and use ML models

**Data pipeline orchestration**
1. Design and implement simple data pipelines
2. Schedule, automate, and monitor basic data processing tasks

**Data management**
1. Configure access control and governance
2. Configure lifecycle management
3. Identify high availability and disaster recovery strategies for data in Cloud Storage and Cloud SQL
4. Apply security measures and ensure compliance with data privacy regulations

Concepts to understand:
- Data manipulation methodologies: ETL, ELT, ETLT
- Data transfer types
- Data quality and cleaning
- Data formats: CSV, JSON, APache Parquet, Apache Avro, structured db tables
- Data storage locations: Regional, dual-regional, multi-regional, zonal
- Data use cases by data requirements: Structured, unstructured, semi-structured
- ML project planning: Data collection, model training, model evaluation, prediction
- SQL for creating, training, and evaluating models
- Data pipeline types
- Data access control and governance: Least privileged
- Backup and recovery solutions, replication
- Encryption in use, in transit, and at rest

Tools to understand:
- Data cleaning: Cloud Data Fusion, BigQuery, SQL, Dataflow
- Data extraction: Dataflow, BigQuery Data Transfer Service, Database Migration Service, Cloud Data Fusion
- Data storage: Cloud Storage, BigQuery, Cloud SQL, Firestore, Bigtable, Spanner
- Data loading: gcloud and BQ CLI, Storage Transfer Service, BigQuery Data Transfer Service, client libraries, Pub/Sub
- Data analysis and visualization: BigQuery, Colab Enterprise (Jupyter notebooks), Looker/Looker Studio/LookML
- Machine learning: BigQuery ML training, evaluation, and inference; Auto ML; pretrained BigQuery; Model Registry
- Data transformation: Dataproc, Dataflow, Cloud Data Fusion, Cloud Composer, Dataform
- Data pipeline automation: BigQuery, Cloud Scheduler, Cloud Composer; Cloud Logging, Cloud Monitoring
- Data orchestration: Cloud Composer, scheduled queries, Dataproc Workflow Templates, Workflows

- Data governance: Identity and Access Management (IAM) roles, Cloud Storage access control, Analytics Hub data sharing
- Data lifecycle management: Cloud Storage classes based on frequency and retention, automated data removal with BigQuery, Cloud Storage
- Primary and secondary data storage locations for data redundancy
- Data privacy: customer-managed encryption keys (CMEK), customer-supplied encryption keys (CSEK), Google-managed encryption keys (GMEK); Cloud Key Management Service (Cloud KMS) to manage keys

## Self-study
**BigQuery**
Fully managed PaaS solution to manage and analyze data.
- Built-in features include ML, search, geospatial analysis, BI
- Uses a serverless architecture, no infrastructure management needed
- Works with structured and unstructured data, supports open table formats
- BigQuery streaming supports continuous data ingestion and analysis
- Has a scalable, distributed analysis engine for querying and analytics needs 
- Built-in data governance, features like: semantic search, data lineage; Dataplex Universal Catalog is the central governance solution
- Architecture in 2 parts
    - Storage layer: ingests, stores, and optimizes data
    - Compute layer: analytics capabilities

BigQuery storage:
- Data in BigQuery is stored in a columnar format optimized for analytical queries
- Data presented in tables, rows, and columns; full support for ACID semantics
- Automatically replicated across locations to provide high availability

BigQuery analytics:
- Descriptive and prescriptive analysis: BI, ad hoc, geospatial, and ML

BigQuery administration:
- BigQuery provides centralized management of data and compute
- IAM helps secure resources with the access model used throughout Google Cloud

**Google Cloud Storage**
Service for storing data objects in Google Cloud. 
- Hierarchy: Organization > Project > Bucket > Folder > Object
    - Objects are defined as immutable data of any format
    - Objects are stored in containers called buckets
    - Can optionally organize objects in buckets into folders
    - All buckets are associated with a project
    - Projects can be grouped under an organization
- Every project, bucket, managed folder, and object is a resource

Cloud Storage tools:
- Console: Visual interface to manage Cloud Storage data in a browser
- gcloud CLI: Interact with Cloud Storage through a terminal using gcloud storage commands
- Client libraries: Allow data management using your preferred programming language
- REST APIs: Manage data using the JSON or XML API
- Terraform: Provision Cloud Storage infrastructure using IaC
- gRPC: Interact with Cloud Storage and define services using Protocol Buffers
- Cloud Storage FUSE: Mount Cloud Storage buckets to your local file system

Cloud Storage security:
- IAM
- Data encryption
- Authentication
- Soft delete
- Object versioning
- Bucket IP Filtering
- Bucket Lock

**Dataflow**
Service providing a unified stream and batch data processing at scale. 
- Create data pipelines to read data from sources, transform data, and write data to a destination

Typical use cases:
- Data movement
- ETL
- Backend support for BI dashboards
- Real-time ML analysis of streaming data
- Sensor data processing or log data processing at scale

Data processing features:
- Fully managed data processing
- Scalable batch and streaming pipelines
- Pipelines are portable, built with Apache Beam
- Data pipeline development is flexible, from straightforward use cases to advanced applications
- Provides monitoring of data pipeline jobs

Job types:
- Runner: Job of running the pipeline on a specific platform

Typical ETL and BI solution using Dataflow and other Google Cloud services:
1. Pub/Sub ingests data from an external system
2. Dataflow reads data from Pub/Sub, writes it to BigQuery; Dataflow might transform or aggregate the data
3. BigQuery acts as a data warehouse, allows for ad hoc querying of the data
4. Looker provides real-time BI insights from the data stored in BigQuery

## Study: Google Learning Path
**Role of a data engineer**
- At a basic level, data engineers build data pipelines
    - Pipelines allow data to get to a place where data-driven decisions can be made
    - Raw data is not useful until processed
- Main tasks
    - Ingestion and storage: Get the data to where it can be useful
    - Data transformation: Get the data into a usable condition
    - Data provisioning and enrichment: Add new value to the data
    - Security, privacy, discovery, governance: Manage the data
    - Pipeline monitoring and automation: Productionize data processes
- Data pipeline
    - Replicate and migrate: Transfer data into GCP
    - Ingest: Raw data made available as a data source
    - Transform: Process data using EL, ELT, or ETL tools
    - Store: Processed data made available in a data sink

**Data source vs. data sink**
- Interstage of a data pipeline, where data becomes a data source
- Data source is raw, unprocessed, but available to be used
- Any platform that stores data can be a data source
    - Cloud Storage
    - Pub/Sub: Async messaging system
- Data sink, where processed data is stored and available
    - BigQuery: Serverless data warehouse
    - BigTable: Scalable NoSQL database

**Data formats**
- Unstructured: Documents, images, audio files
- Structured: Tables with rows and columns

**GCP storage solutions**
- Unstructured data uses Cloud Storage 
    - Benefits
        - Reliability and scalability
        - Accessed by HTTP request
        - Retrieved by object name
        - Max object size of 5 TB
    - Provides 4 data storage types
        - Standard (real-time access)
        - Nearline (monthly access)
        - Coldline (quarterly access)
        - Archive (yearly access)
- Many structured data storage services
    - Cloud SQL: Managed RDBS
    - AlloyDB: PostgreSQL service
    - Spanner: Fully-managed RDBS
    - Firestore: Fully-managed serverless NoSQL database
    - BigQuery: Fully-manged serverless 
    - BigTable: Fully-manged serverless table service
- Cloud storage service types
    - Data lake: Stores raw data for future data use
        - Relies on other tools to enable higher level functions
        - Ex. Cloud Storage
    - Data warehouse: Stores pre-processed data from multiple data sources
        - Standalone service, allows for long-term business analysis 
        - Ex. BigQuery
- BigQuery: Serverless, fully managed data warehouse
